# 用户自定义规则 (模板)

> 在此添加您希望 AI 遵守的项目规则

## 代码规范

<!-- 示例规则，请根据您的项目修改 -->

- 命名规范：使用驼峰命名法 (camelCase)
- 函数长度：每个函数不超过 50 行
- 注释要求：公共函数必须有文档字符串
- 类型注解：建议但不强制

## 交互规范

<!-- 定义项目的 UI 交互规则 -->

- 保存模式：设置页面使用实时保存，表单使用按钮保存
- 输入控件：选项超过 5 个时使用下拉菜单
- 确认操作：删除等破坏性操作需要二次确认
- 反馈机制：操作完成后显示 Toast 提示

## 架构规范

<!-- 定义项目的代码组织规则 -->

- UI 分离：界面代码与业务逻辑严格分离
- 模块化：功能模块尽可能独立，便于复用
- 配置集中：配置项统一放在 config 目录

## 修改规则

<!-- 特别重要的修改规则 -->

- 修改函数签名时，必须同步修改所有调用处
- 修改数据结构时，必须检查所有使用该结构的代码
- 删除功能时，必须清理所有相关的引用和资源
